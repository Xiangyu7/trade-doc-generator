外贸单据 RPA —— 使用说明
================================

【两种用法，任选】

★ 用法一：网页版（推荐，拖拽上传，最直观）
  1) 在「单据自动生成」文件夹里打开“终端”，运行：
         python3 app.py
  2) 浏览器会自动打开 http://127.0.0.1:5000
  3) 页面有三块功能：
     ① 生成发票(PI/CI/PL)：把客户 PO 拖进左边框 → 自动预填表单
        → 核对/修改（尤其品名、单价、数量）
        → 勾选这次要出的单据（PI 形式发票 / CI 商业发票 / PL 装箱单，默认全选）
        → 点“生成所选单据” → 每种单据各得一份独立的 Excel + PDF，分别下载
        （CI/PL 靠模板公式联动主表，单独导出也会自动取到数据）
        （扫描件/图片 PO 会自动用 OCR 识别，首次识别要先编译一次约15秒；
         数量会自动按“吨×1000”换算成公斤、单价÷1000。没有 PO 也可直接手填）
     ② 合并清关包：把提单/保险单/COA/产地证等拖进右边框（可多选）
        → 用 ↑↓ 调顺序、✕ 删除 → 点“合并” → 下载一个清关 PDF
        （支持 PDF / Excel / Word / 图片jpg,png，图片会自动转成PDF页）
     ③ 危化品文档：可把「性能单 / 分类鉴定报告」拖进上方框自动提取部分字段
        （性能单→性能单号/UN标记；鉴定报告扫描件走OCR→报告编号/UN编号，务必核对）
        → 选「包装类型」(200kg桶装 / 吨桶，切换会自动换厂检单字段默认值)
        → 核对修改通用字段 + 厂检单字段 → 点“生成危化品单据”
        → 出 符合性声明 / 责任声明 / 对应包装的厂检单（各 Word + PDF）
  4) 用完在终端按 Ctrl+C 停止。
  注：已接入 MiniMax M3 做 AI 识别。识别 PO 时，PO 的文字内容会发送到
      MiniMax 服务器(api.minimaxi.com)做字段提取——这一项不再是纯本地。
      其余功能(合并清关、危化品生成、导出PDF)仍全部在本地完成，文件不外传。
      想完全离线：删掉或清空「单据自动生成/.ai_key」文件，会自动回退本地正则识别。

★ 用法二：批量命令行版（一次生成很多票）
  1) 打开「订单信息表.xlsx」，每行填一票订单（发票号必填，没填的行会跳过）。
     多行内容（买方地址/品名+HS/通知方）在格子里按 Alt+Enter 换行。
  2) 终端运行：  python3 生成单据.py
  3) 结果在「输出单据」文件夹。

【文件说明】
  app.py          —— 网页后端（用法一）
  templates/      —— 网页前端页面
  core.py         —— 核心逻辑（生成发票/合并PDF/识别PO），两种用法共用
  ai_extract.py   —— AI 识别模块（MiniMax M3），core.py 识别 PO 时调用
  .ai_key         —— MiniMax API key（明文，注意保密；换号直接覆盖此文件）
  生成单据.py      —— 批量命令行版（用法二）
  订单信息表.xlsx  —— 批量版填这个（已预填3行示例）
  输出单据/        —— 所有生成结果
  uploads/        —— 网页上传的临时文件（可随时清空）

【订单字段对应单据上的位置】
  发票号→Invoice No.  发票日期→PI Date.  ReferenceNo→Reference No.
  买方名称及地址→Buyer/Consignee   通知方→Notify Party
  品名及HS编码→Goods Description   数量KGS→Quantity   单价USD→Unit Price
  （金额自动 = 数量×单价）
  付款条款→Terms of Payment   贸易术语→Terms of delivery   包装方式→Packing
  装货港→Loading Port   卸货港→Discharge port   唛头→Shipping Mark
  体积CBM→装箱单 Measurement

【原理 / 注意】
  - 只填“形式发票”主表，商业发票(CI)和装箱单(PL)靠模板里的公式自动联动。
  - 卖方抬头、银行信息、装箱单抬头都是模板里固定的（在你自备的 CPH模板.xlsx 里）。
  - 改格式/字段：直接改主文件夹的「CPH模板.xlsx」，工具会跟着变。
  - PO 自动识别是“尽力而为的草稿”，一定要人工核对（尤其单价/数量/金额/品名）。
    识别流程：先读 PDF 文字层（扫描件/图片用 macOS 自带 OCR，ocr_helper.swift
    首次自动编译成 ocr_helper）→ 再把文字交给 MiniMax M3 抽取成字段。
    · AI 配置：key 放在「单据自动生成/.ai_key」；模型/接口可用环境变量
      MINIMAX_MODEL、MINIMAX_BASE_URL 覆盖（默认 MiniMax-M3 / api.minimaxi.com）。
    · 没配 key 或断网，自动回退本地正则识别，工具照常能用（命中率略低）。
    · AI 也可能看错，务必逐项核对。
  - 导 PDF 用 LibreOffice（已安装）。
